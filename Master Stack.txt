4. Smooth Operating Rules

To keep the architecture consistent, use these rules:

PostgreSQL is the source of truth

FastAPI handles business logic and service APIs

React/Next handles presentation

Redis handles speed, caching, queue support

Microservices only exist when a function is clearly separate

Hydra/MCP handles documentation, task orchestration, and AI-assisted workflows

Every system gets docs, schema notes, and changelog discipline

Shared design system keeps brand/UI consistent across products

Analytics engines should be isolated from frontend concerns

Content systems should reuse the same metadata taxonomy as product docs when possible

7. Best-Practice Consistency Rules

Use the same naming pattern everywhere:

apps/ for full user-facing products

services/ for backend or microservices

packages/ for shared utilities/components/types

docs/ for internal system documentation

data/ for seeds, migrations, schemas, raw imports

infra/ for Docker, deployment, CI/CD


Use the same logic pattern everywhere:

ingest -> validate -> store -> calculate -> expose -> visualize


Use the same documentation pattern everywhere:

overview

goals

schema

API

workflows

task list

changelog

deployment notes
