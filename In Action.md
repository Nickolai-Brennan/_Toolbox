1️⃣ IDENTIFY THE NEED (Trigger Layer)

Goal: Define why this exists before doing anything else

Inputs

User pain point

Internal inefficiency

Market gap

Feature request


Output Doc

PROBLEM_STATEMENT.md

USER_NEED_WORKFLOW.md


Structure

Problem

Who experiences it

When it occurs (workflow stage)

Current workaround (if any)

Impact (time, money, confusion)


Rule If the need is unclear → stop. No execution.


---

2️⃣ POST USER WORKFLOW NEED (Clarity Layer)

Goal: Visualize the exact user journey

Output Doc

USER_WORKFLOW.md


Format

Step-by-step flow (no assumptions)


Example:

User enters platform

Searches for X

Fails to find Y

Leaves or hacks workaround


Add

Friction points

Drop-off points

Confusion areas


Key Insight You are not solving a feature → you are fixing a workflow.


---

3️⃣ BRAINSTORM SOLUTIONS (Exploration Layer)

Goal: Expand solution space before locking in

Output Doc

SOLUTION_BRAINSTORM.md


Break into 3 categories

Simple Fix (fastest MVP)

System Fix (proper solution)

Scalable Fix (future-proof)


For each idea

Description

Pros

Cons

Effort level

Dependencies


Rule Minimum 5 solutions before choosing


---

4️⃣ SELECT & DEFINE SOLUTION (Decision Layer)

Goal: Lock scope → prevent scope creep

Output Doc

SOLUTION_DECISION.md

FEATURE_SPEC.md (PRD-lite)


Define

What it does

What it does NOT do

Inputs / Outputs

User interaction

Edge cases


Add

Success metric (critical)

Example:

“User completes task in < 2 min”

“Search success rate > 80%”




---

5️⃣ DEVELOP THE SOLUTION (Execution Layer)

Goal: Build only what was defined

Docs Created

TECH_SPEC.md

API_SPEC.md (if needed)

UI_SPEC.md

DB_MAPPING.md


Execution Breakdown

Frontend tasks

Backend tasks

Data layer tasks

Integration tasks


Tracking

Every task → checklist item (Master Control File rule)



---

6️⃣ VALIDATE & TEST (Quality Layer)

Goal: Ensure it actually solves the problem

Output Docs

TESTING_PLAN.md

VALIDATION_REPORT.md


Test Types

Functional (does it work)

Workflow (does it solve problem)

Edge cases

Speed / performance


Critical Test against original workflow — not assumptions


---

7️⃣ DOCUMENT THE PROCESS (System Layer)

Goal: Turn solution into reusable asset

Output Docs

PROCESS_DOC.md

HOW_IT_WORKS.md

FUTURE_IMPROVEMENTS.md


Include

Before vs After workflow

Architecture overview

Decisions made (why)
